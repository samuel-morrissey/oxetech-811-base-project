# Checkpoints do Projeto Integrador

Este documento transforma a proposta avaliativa em um checklist pratico para evoluir a mesma codebase ao longo do curso.

## Avaliacao 1

Foco: Clean Code, code smells, refatoracao, SOLID introdutorio e patterns basicos.

Entrega esperada:

- Diagnostico geral dos problemas percebidos na codebase.
- Refatoracoes pequenas e seguras.
- Melhoria de nomes, funcoes e responsabilidades.
- Reducao de duplicacoes simples.
- Aplicacao de pelo menos um principio SOLID, quando fizer sentido.
- Aplicacao de um pattern basico, se o contexto justificar.
- Teste simples ou roteiro de validacao manual.
- Explicacao objetiva no Pull Request.

Nao e esperado:

- Reescrever a aplicacao inteira.
- Criar arquitetura completa.
- Cobertura ampla de testes.
- Docker, CI ou deploy robusto.

## Avaliacao 2

Foco: organizacao arquitetural introdutoria, testes, validacao e seguranca basica.

Entrega esperada:

- Diagnostico atualizado.
- Separacao mais clara entre entrada HTTP, regra de negocio e persistencia.
- Melhoria de pelo menos um fluxo relevante.
- Testes unitarios e/ou de integracao para partes importantes.
- Validacao de entrada em pontos necessarios.
- Tratamento de erros mais consistente.
- Correcao de problemas basicos de seguranca.
- Justificativa das decisoes no Pull Request.

Nao e esperado:

- DDD avancado.
- Clean Architecture completa.
- Mensageria.
- Docker obrigatorio.
- Pipeline CI obrigatorio.
- Deploy sofisticado.
- Observabilidade avancada.
- Cobertura total de testes.

## Projeto Final Integrador

Foco: consolidacao da evolucao tecnica da mesma codebase.

Entrega esperada:

- Historico de Pull Requests preservado.
- Codigo refatorado em partes relevantes da aplicacao.
- Organizacao geral melhor que a inicial.
- Testes automatizados para comportamentos importantes.
- Docker funcional.
- Pipeline CI verde.
- README completo e objetivo.
- Documentacao das principais decisoes.
- Relatorio final de evolucao.

O relatorio final deve cobrir:

- estado inicial da codebase;
- principais problemas encontrados;
- melhorias implementadas;
- conceitos do curso aplicados;
- decisoes tecnicas tomadas;
- evidencias de funcionamento;
- limitacoes conhecidas;
- proximos passos possiveis.

## Criterio central

A maturidade da entrega sera avaliada pela clareza, consistencia e correcao das decisoes, nao pela quantidade de tecnologia adicionada.
